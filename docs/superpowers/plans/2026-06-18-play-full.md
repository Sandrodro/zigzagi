# Play View (Full) — Plan Outline

> **Fidelity: OUTLINE.** Task-level only — files, interfaces, and named tests. Expand each task
> to bite-sized (failing test → run → implement → run → commit) before building, following the
> Walking Skeleton plan's style. Builds directly on the Walking Skeleton's `<Grid>`,
> `CrosswordEngine`, `PlayView`, and the server-side check/reveal endpoints.

**Goal:** Turn the skeleton's bare grid into the NYT-app-like daily solve experience:
on-screen Georgian keyboard, clue bar + across/down lists, timer, square/word/puzzle
check/reveal, congrats, local persistence, and a calm animated background — mobile-first.

**Tech Stack:** React + TypeScript + Vitest (existing). Adds a WebGL fragment shader for the
background. All solving state stays in the pure `CrosswordEngine` (extended), so every behavior
is unit-testable without React.

## Global Constraints (from DESIGN.md §3, §4.8, §5, §9)

- **Reusable components** (`<GeorgianKeyboard>`, `<ClueBar>`, `<ClueList>`, `<Timer>`,
  `<CongratsModal>`); solving logic stays in the engine module, not the components.
- **Answers never shipped** — check/reveal continue to go through the server endpoints.
- **Persistence** in `localStorage` keyed by `date` + a generated `client_id`, **same shape as
  server `Progress.fills`** for clean later merge (Auth plan depends on this shape).
- **Background:** single WebGL shader, low contrast, slow; **hard-disabled** under
  `prefers-reduced-motion` and a settings toggle; **CSS static-gradient fallback** if WebGL is
  unavailable. Budget ≥ 50 fps; never hurts readability.
- **Mobile:** custom on-screen keyboard is the primary input; test iOS + Android early.

## Task List

1. **Extend `CrosswordEngine`: clue model + completion.**
   - Interfaces: `currentClue(): ClueRef`, `clueForCell(row,col,dir): ClueRef`,
     `nextClue()/prevClue()`, `isComplete(): boolean`, `cellsForScope("square"|"word"|"puzzle"): Cell[]`.
   - Tests (`crossword.test.ts`): current clue tracks active cell+direction; `isComplete` true
     only when every playable cell has a value; scope cell-sets correct.

2. **`<GeorgianKeyboard>` component.**
   - Interface: `GeorgianKeyboard({ onKey, onBackspace, onEnter })` — full Mkhedruli layout
     (U+10D0–U+10FF) + backspace + enter; `data-key` on each button.
   - Tests: clicking a letter button calls `onKey("ა")`; backspace calls `onBackspace`.

3. **`<ClueBar>` + `<ClueList>`.**
   - Interfaces: `ClueBar({ clue, onPrev, onNext, direction, onToggleDirection })`;
     `ClueList({ across, down, activeNumber, onSelect })`.
   - Tests: active clue highlighted; selecting a clue calls `onSelect(number, direction)`.

4. **`<Timer>` + timer state.**
   - Interface: `useTimer()` hook (`seconds`, `start`, `pause`, `reset`); persists elapsed.
   - Tests: ticks while running; pauses; survives reload (reads persisted seconds).

5. **Local persistence layer.**
   - Interface: `progress/local.ts` — `loadProgress(date): {fills, timerSeconds, completedAt} | null`,
     `saveProgress(date, state)`, `getClientId(): string` (generated once, stored). Shape **must
     match server `Progress.fills`**.
   - Tests: round-trip; `client_id` stable across calls; keyed per date.

6. **Wire persistence + keyboard + clues into `PlayView`.**
   - On every engine mutation, persist; on mount, hydrate from `localStorage`. Keyboard drives
     `engine.type/backspace`; clue list/bar drive `engine.setActive` + direction.
   - Tests (`PlayView.test.tsx`): typing on the on-screen keyboard fills the grid and persists;
     reload restores fills + timer.

7. **Check/Reveal full UI (square/word/puzzle scopes).**
   - Buttons call `checkCells`/`revealCells` with `engine.cellsForScope(scope)`; results applied
     via `engine.applyCheck/applyReveal`.
   - Tests: word-scope check posts exactly the current word's filled cells; reveal-puzzle fills all.

8. **`<CongratsModal>` on completion.**
   - Shows when `engine.isComplete()` and a full puzzle-check returns all-correct; displays final
     time. Sets `completedAt` in persisted progress.
   - Tests: appears only on all-correct completion; shows the timer value.

9. **WebGL hazy background + reduced-motion + toggle + fallback.**
   - Interface: `<Background enabled />` — WebGL shader layer; respects
     `window.matchMedia("(prefers-reduced-motion: reduce)")` and a settings toggle; falls back to
     a CSS gradient when WebGL context creation fails.
   - Tests: renders gradient fallback when WebGL unavailable (mock `getContext` → null); renders
     nothing animated when reduced-motion is set.

10. **Mobile pass (manual + E2E).**
    - Manual: iOS Safari + Android Chrome — Georgian on-screen keyboard, no native-keyboard
      popup stealing focus, tap targets ≥ 44px, no layout shift from the background.
    - Deliverable: documented device matrix results; zero blocking bugs in the solve loop.
